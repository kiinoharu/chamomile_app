## サービス開発の経緯
最近、私の身内に子宮頸がんの検診にて問題が見つかりました。幸い早期発見できたことから、経過観察にて様子を見る状況となっています。<br>
私はこの経験から、世の中の特に女性に対して、子宮頸がんや婦人系疾患に関する知識や理解を普及したいと考えるようになりました。<br>
そこで、生理周期管理アプリであれば、婦人系疾患に悩むユーザー層が多いことや婦人系疾患とのシナジーがあることから、最適なプラットフォームであると感じました。そして、**生理周期管理機能**に加えて、**婦人系疾患情報閲覧機能**を付与することで、婦人系疾患に関する知識や理解が、ユーザーへ少しでも普及でき、病気へのリスク管理をされる女性が増加すると考え、Chamomile-appを開発しました。<br>
カモミールは、リラックス効果や痛みを和らげる効果があることから、生理周期管理アプリ名に合っていると思い、命名しました。

## 今後の展望
以下の項目について実装していき、ユーザーへの快適な操作性や必要な価値を提供できるようにしていく予定です。
<br><br>
#### ・ユーザー機能の拡張
　現状「user_id:1」のユーザーのみの記録しか登録できない仕様になっているため、新規登録機能からuser管理に関する実装を行い、画面内容も各ユーザーのDB情報とリンクする形で表示するように実装します。<br>
#### ・アナウンス機能の実装
　各ユーザーの生理周期を取得し、生理周期に±3日以上のずれが発生した際に、婦人系疾患の予兆の可能性を鑑み、アナウンス機能を追加し、早期受診を促すことを目的とした機能を付与していきます。<br>
 また、不正出血が生理周期を除く形で連続して1週間以上記録された場合にも上記同様のアナウンス機能を付与します。<br>
#### ・掲示板へ投稿機能実装
　掲示板画面において、ユーザーの体験や経験を投稿できる機能を実装します。目的としては、人それぞれの経験を共有することで、各ユーザーの婦人系疾患に対する意識をより向上させることです。<br>
 　また、人それぞれの様々な経験を閲覧することで、同様の事象で悩まれている方の助けになることがあるかもしれないと考え、実装をしていく方針です。
